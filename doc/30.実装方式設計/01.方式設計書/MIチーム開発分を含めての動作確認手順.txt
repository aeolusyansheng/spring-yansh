■ 準備（一度だけ行う手順）

1. CTCから送付されたソース一式を、作業用のフォルダ（例 C:\tmp）に解凍する。
2. Eclipse を起動する。（これまで通りの開発時の状態）
2. Eclipse のメニューで[ファイル]-[ワークスペースの切り替え]-[その他]を選ぶ。
3. ワークスペースとして「../workspace-all」を指定し、コピー設定のチェックボックス2つをONにしてOKボタンを押す。
4. Eclipse が自動的に再起動するので、起動完了まで待つ。
5. Windowsのエクスプローラで、新しいワークスペースのフォルダ（C:\pjnavi\pleiades-e3.7-java_20120225_for_pjnavi\workspace-all）ができているのを確認する。
6. Eclipse のメニューで[ウィンドウ]-[設定]を選ぶ。
7. [サーバー]-[ランタイム環境]を選び、右側の[追加]ボタンを押す。
8. [Apache]-[Apache Tomcat v7.0]を選び、[次へ]ボタンを押す。
9. [Tomcatインストールディレクトリ]に「C:\pjnavi\pleiades-e3.7-java_20120225_for_pjnavi\apache-tomcat-7.0.32」を指定する。
10. [JRE]に「jdk1.7.0_09」を指定し、[終了]ボタンを押す。
11. [OK]ボタンを押して設定ウィンドウを閉じる。
12. Eclipse のメニューの [ファイル]-[インポート]を選ぶ。
13. [一般]-[既存プロジェクトをワークスペースへ]を選んで[次へ]ボタンを押す。
14. [ルートディレクトリの選択]で、作業用フォルダ内の pjnavi-builder（例 C:\tmp\pjnavi-builder）を指定する。
15. [プロジェクトをワークスペースにコピー]をONにし、[終了]ボタンを押す。
16. 12〜15と同様の手順で、下記の各プロジェクトをインポートする。
      ・pjnavi-fw
      ・pjnavi-cm
      ・pjnavi-batch
      ・pjnavi-mobile
      ・pjnavi-server
      ・pjnavi-client-fw
      ・pjnavi-client-cm
      ・pjnavi-client
   なお、ビルド中に下記のようなエラーメッセージが出るが、動作には影響ないので無視して良い。
      ・プロジェクト 'pjnavi-***' でビルダー 'Checkstyle ビルダー' の実行中にエラーが発生しました
      ・/pjnavi-***/pjnavi-tool/settings/pjnavi-checks.xml を見つけることができませんでした
17. pjnavi-cm/src/jp/co/ctc_g/pjnavi/sy/ldap/LdapServiceImpl.java を開く。
18. 57〜69行目の search メソッドの内側を全てコメントアウトし、代わりに下記のコードを書いて保存する。
      return Arrays.asList();   // LDAPサーバへの問い合わせをスキップする
19. pjnavi-server/resource/Development-Context.xml を開く。
20. 63行目（otlwsProxy）がコメントアウトされているのを解除（コメントの外に出す）して保存する。
21. Eclipse の[サーバー]ビューを右クリックして、メニューから[新規]-[サーバー]を選ぶ。
22. 「Tomcat v7.0 サーバー」を選び、[サーバー名]を「pjnavi-server」に変えて[次へ]ボタンを押す。
23. 左側の「pjnavi-server」をダブルクリックし、[終了]ボタンを押す。
24. Eclipse の[サーバー]ビューを右クリックして、メニューから[新規]-[サーバー]を選ぶ。
25. 「Tomcat v7.0 サーバー」を選び、[サーバー名]を「pjnavi-mobile」に変えて[次へ]ボタンを押す。
26. 左側の「pjnavi-mobile」をダブルクリックし、[終了]ボタンを押す。

■ pjnavi-server の動作確認手順

これまでと同様。

■ pjnavi-mobile の動作確認手順

1. Eclipse の[サーバー]ビューで「pjnavi-server」を右クリックし、[デバッグ]を選ぶ。
2. Tomcatが起動するのを待つ。
3. FirefoxなどのWebブラウザを開き、下記のURLにアクセスする。パラメータ「empcd」の値は、DB内の従業員マスタに存在する従業員コードにprefix "z" を付けたものにする。
   http://localhost:8080/pjnavi-mobile/dmiotl13000?empcd=***&password=x
4. Webブラウザに[稼働日選択]画面が表示されればOK。エラー画面が表示された場合は、Eclipse の[コンソール]に出力されているSQLを参考に、必要なマスタデータを投入して再度試す。
   （ボタンマスタでイベントURI "/rest/mi/otl/dmiotl02000/write" に紐付く職責が必要）
